# Claude Code 입문 학습 정리

> 작성일: 2026-02-23

## 핵심 개념

- Claude Code는 터미널에서 AI와 대화하며 코딩하는 도구
- 일반 말로 명령하면 Claude가 파일 읽기/쓰기, Git 작업 등을 수행
- 세션 시작 시 `CLAUDE.md`를 자동으로 읽어 프로젝트 맥락을 파악

## 슬래시 커맨드

| 커맨드 | 역할 |
|--------|------|
| `/status` | 현재 버전, 모델, 메모리 설정 확인 |
| `/rename` | 세션 이름 지정 |
| `/hooks` | Hook 설정 메뉴 열기 |
| `/checkpoint` | 현재 상태 저장 |
| `/rewind` | 저장 시점 선택 후 되돌리기 |
| `/memory` | 기억 내용 확인/편집 |
| `/clear` | 대화 기록 초기화 (CLAUDE.md는 유지) |

## @ 파일 참조

```
@파일경로 를 프롬프트에 쓰면 Claude가 해당 파일을 자동으로 읽음

예시:
@CLAUDE.md 파일을 읽고 개선점을 제안해줘
@실밸개발자/ep01.md 요약해줘
```

## CLAUDE.md

- 세션 시작 시 자동으로 로드되는 프로젝트 지침 파일
- Claude의 동작 방식, 컨벤션, 규칙 등을 명시
- `/status`에서 `Memory: project (CLAUDE.md)` 로 로드 여부 확인 가능

## Hooks

특정 순간에 자동으로 쉘 명령어를 실행하는 자동화 규칙

| Hook | 시점 | 활용 예 |
|------|------|---------|
| `PreToolUse` | 도구 사용 전 | 위험한 파일 수정 차단 |
| `PostToolUse` | 도구 사용 후 | 자동 포맷팅 |
| `PostToolUseFailure` | 도구 실패 후 | 에러 로깅 |
| `Notification` | Claude 입력 대기 중 | 데스크톱 알림 |
| `UserPromptSubmit` | 명령어 제출 시 | 명령어 로깅 |

```bash
# 설정 위치
.claude/settings.json         # 프로젝트용
~/.claude/settings.json       # 전체 사용자용
```

## 커스텀 슬래시 커맨드

`.claude/commands/` 폴더에 마크다운 파일을 만들면 슬래시 커맨드가 됨

```
.claude/commands/note.md   → /note 커맨드
.claude/commands/my-git.md → /my-git 커맨드
```

**파일 형식:**

```markdown
---
description: /help에 표시될 설명
allowed-tools: [Read, Bash]
---

Claude가 따를 지시사항

$ARGUMENTS  ← 사용자가 입력한 인자
!`git status`  ← ! 뒤 쉘 명령어는 실행 결과가 자동 삽입
```

## 실습 메모

- SSH 환경에서는 `Notification` Hook의 `notify-send`가 동작하지 않음
  → SSH 개발 시 알림 Hook보다 `PostToolUse`, `PreToolUse`가 더 유용
- `/rewind`는 특정 시점을 선택하는 메뉴가 나타남 (무조건 직전이 아님)
- `@파일참조`는 가장 자주 쓰게 되는 기능 중 하나
- 커스텀 커맨드는 frontmatter 없이도 동작함 (`my-git.md` 참고)

## MCP (Model Context Protocol)

Claude에 외부 도구를 연결하는 플러그인 시스템. 사용할 때만 토큰 소비 (고정 비용 없음)

```bash
# PAT 방식으로 GitHub MCP 추가 (SSH 환경에서 추천)
claude mcp add --transport stdio github \
  --env GITHUB_PERSONAL_ACCESS_TOKEN=토큰 \
  -- npx -y @modelcontextprotocol/server-github

claude mcp list    # 목록 및 연결 상태 확인
claude mcp remove  # 제거
```

**저장 수준:**
- `local` — 현재 프로젝트, 개인용 (기본값)
- `project` — `.mcp.json`에 저장, 팀 공유
- `user` — 모든 프로젝트에 적용

**실습 메모:**
- HTTP 방식(`https://api.githubcopilot.com/mcp/`)은 OAuth 필요 → SSH에서 불편
- PAT stdio 방식이 SSH 환경에서 더 안정적
- Claude가 `gh` CLI 실패 시 자동으로 MCP 도구로 전환하는 것을 확인
- MCP 응답은 구체적으로 요청할수록 토큰 절약 ("모든 ~" 보다 "최근 5개 ~")
- **PAT 토큰은 절대 대화창에 입력하지 말 것** (노출 위험)
